System Architecture

Block diagram of the system architecture

Smart Sense Sensors

The sensor is responsible for recording the environmental data from the location it is installed. The sensor uses an Arduino Genuino 101 board for its processor and its onboard BLE capabilities. The sensors attached to the Genuino include an analogue light sensor and a DHT11 sensor which is a digital sensor that records temperature and humidity. An LED is also attached to the board which is the light that can be controlled from the cloud platform.





The Smart Sense Sensor

Smart Sense Gateway

The gateway operates two-way communication between the sensor and the MQTT broker. The gateway is connected to the sensors over a BLE connection and to the MQTT broker over internet. Connection to the MQTT broker is secured by a username and password which ads a layer of security. When lighting control commands are sent from the MQTT broker, the gateway passes these to the sensor so that the LED can be controlled. It also reads sensor values from the sensors, aggregates these values to avoid unwanted ‘spikes’ or errors and then sends the data in JSON form to the MQTT broker. The gateway operates on a Raspberry Pi 3 running Raspbian which is a Linux based operating system and it uses the Node and Noble library for its operation.



The Smart Sense Gateway

MQTT Broker

The MQTT broker provides two-way communication between the gateway and the cloud platform. The server has four topics: temperature, humidity, light level and light control to which messages can be published. These messages can be retrieved by subscribing to the relevant topic. Sensor data is published by the gateway to temperature, humidity and light level topics where it can then be accessed by the database. Lighting control commands are sent directly from the user interface to the light level topic, where then can then be accessed by the gateway.

MongoDB Server

The MongoDB database stores the sensor data in the form of JSON objects that are published to the MQTT topics. There are three collections within the smartsense database: temperature, humidity and light level each of these stores the corresponding sensor data. This database is hosted on the cloud and can be accessed through a RESTful API.

Cloud Platform

The Cloud Platform is the client-side server that displays data to the user and enables control of the lights attached to each sensor, it is accessed through the URL ‘https://smartsense.ie/index.html’. The web content which makes up the website is stored in an AWS S3 bucket which is configured to run as a static website. Access to the website is routed though AWS CloudFront which improves load time though cleaver caching of pages already loaded into AWS ‘edge zones’. CloudFront also applies an SSL cert to the website which was authorised by AWS Certificate Manager, a service that provides Secure Sockets Layer/Transport Layer Security (SSL/TLS) certificates for use with AWS services. This ensures that network communication to and from the website is encrypted.
